Git – Sistemul de Control al Versiunilor

Git este un sistem distribuit de control al versiunilor (DVCS), creat de Linus Torvalds în 2005 pentru dezvoltarea kernel-ului Linux. Este folosit de peste 95% dintre dezvoltatorii profesioniști din lume.

DVCS

Ce este Controlul de Versiune?

Un sistem de control al versiunilor (VCS) permite:

  • Înregistrarea fiecărei modificări în fișiere
  • Revenirea la versiuni anterioare
  • Colaborarea între mai mulți dezvoltatori
  • Urmărirea autorului fiecărei modificări
  • Prevenirea conflictelor și pierderii codului

Arhitectura Git – Cum funcționează?

Git folosește un model de snapshots, nu diferențe. Fiecare commit este o fotografie completă a proiectului la acel moment.

  • .git/ – directorul ascuns care conține toată istoria
  • Obiecte: blob (fișier), tree (director), commit
  • Fiecare obiect are un hash SHA-1 unic

Stadiile unui fișier în Git

  1. Untracked – fișier nou, necunoscut de Git
  2. Staged – adăugat cu git add
  3. Committed – salvat în repository cu git commit

Comenzi esențiale Git – Ghid complet


# Configurare inițială
git config --global user.name "Nume Prenume"
git config --global user.email "email@tudomain.com"
git config --global init.defaultBranch main

# Inițializare și clonare
git init                          # Creează un repo nou
git clone https://github.com/...  # Clonează un repo existent

# Lucru cu fișiere
git status                        # Starea curentă
git add .                         # Adaugă toate fișierele
git add fisier.html               # Adaugă un fișier
git commit -m "Mesaj descriptiv"  # Commit cu mesaj
git log                           # Istoric complet
git log --oneline --graph         # Istoric vizual
git diff                          # Diferențe nemodificate
git diff --staged                 # Diferențe staged

# Ramuri (branching)
git branch                        # Listează ramurile
git branch feature/login          # Creează ramură
git checkout feature/login        # Trece pe ramură
git checkout -b feature/login     # Creează + trece
git merge main                    # Îmbină ramura curentă în main
git branch -d feature/login       # Șterge ramură

# Lucru la distanță
git remote -v                     # Listează remote-uri
git remote add origin URL         # Adaugă remote
git push origin main              # Încarcă pe GitHub
git pull origin main              # Actualizează local
git fetch                         # Descarcă fără a îmbina

# Revenire în timp
git checkout HASH                 # Revine la un commit
git reset --hard HASH             # Revine forțat
git revert HASH                   # Creează commit invers
git stash                         # Salvează temporar
git stash pop                     # Restaurează

Flux de lucru tipic (Git Flow)

  1. main – ramura stabilă, doar cod testat
  2. develop – ramura de integrare
  3. feature/* – ramuri pentru funcționalități noi
  4. release/* – pregătire lansare
  5. hotfix/* – corecturi urgente

Exemplu practic: Crearea unei funcționalități


git checkout main
git pull origin main
git checkout -b feature/autentificare
# ... dezvoltare ...
git add .
git commit -m "Adăugat login cu email"
git push origin feature/autentificare
# → Creezi Pull Request pe GitHub
